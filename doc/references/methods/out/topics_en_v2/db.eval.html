db.eval()

NAME

eval - According need to fill javaScript statement.

SYNOPSIS

db.eval(<code>)

CATEGORY

Sequoiadb

DESCRIPTION

According need to fill javaScript statement. Meanwhile you can a stored procedure in the statement has been created good.

code (string)
JavaScript statement or stored procedure to create a good function.
  • In accordance with the statement executed successfully return results. The return value can be directly assigned to another variable. Eg: var a = db.eval(' db.foo.bar'); a.find();
  • fails returns an error code and error message. Such as: { "errmsg": "(nofile):1 ReferenceError: sum is not defined","retCode": -152 }
  • Operation does not return before the end of the function execution. Dropouts terminated throughout the implementation, but the code has been executed will not be rolled back.
  • Customize the length of the return value of a certain limit, the reference SequoiaDB insert records maximum length.
  • Define the temporary support functions, such as: db.eval(' function sum(x,y){return x+y;} sum(1,2)')
  • Global db usage and createProcedure same.
Note: While all the output statement will be shielded, but it is recommended not to join any print statements.
