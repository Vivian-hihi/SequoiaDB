db.createCataRG()

NAME

createCataRG - Create a new catalog ReplicaGroup.

SYNOPSIS

db.createCataRG(<host>,<service>,<dbpath>,[config])

CATEGORY

Sequoiadb

DESCRIPTION

Create a new catalog ReplicaGroup, and create and start a catalog node.

host (string)
Specifies the host name of the catalog node.
service (int)
Service port specified catalog node, make sure that the port number, and the subsequent continuation of the three port numbers unoccupied, if set to 11800, make sure 11800/11801/11802/11803 ports are not occupied.
dbpath (string)
Data file path used to store catalog data files, administrators need to ensure that the data (created during installation, the default is sdbadmin) the user has write permissions.
config (json)
parameter is optional, for more details on the configuration parameters, the format must be json format, database configuration parameters see a, If you need to configure the log size parameters {logfilesz:64}.
Format
Definition Format createCataRG() method has host, service, dbpath, config four parameters, host, dbpath a string type, service is an integer type, config as json object in the following format:
{"<host name>",<port number>,"<data file path>",[database configuration parameter object]}
Note:
  • Make sure the permissions data file storage path, if SequoiaDB using the default installation, then the path to the authority conferred sdbadmin.

Examples

Named: Create a catalog node group on sdbserver1 host, service port: 11800, data files path: /opt/sequoiadb/database/cata/11800

db.createCataRG("sdbserver1", 11800, "/opt/sequoiadb/database/cata/11800")
