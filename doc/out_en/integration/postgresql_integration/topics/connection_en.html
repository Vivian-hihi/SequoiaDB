Integration with 3rd party components / Integration with PostgreSQL

Connection

The following operations are all under PostgreSQL's shell panel.

Mapping SequoiaDB to PostgreSQL

1) Load SequoiaDB's connecting driver

foo=# create extension sdb_fdw;

2) Set the parameters to connect SequoiaDB

foo=# create server sdb_server foreign data wrapper sdb_fdw options(address '127.0.0.1', service '11810', user 'sdbadmin', password 'mypassword');
Note: If there is no password verification, you can ignore field user and password.

3) Mapping SequoiaDB' scollection space and collection

foo=# create foreign table test (name text, id numeric) server sdb_server options ( collectionspace 'chen', collection 'test' ) ;
Note:

The collection space and collection must exist in SequoiaDB database created.

By default, tables of PG are mapped to SequoiaDB as lower case letters, if you want to set it as capital, please see the Notice later in this page.

The mapping of array type in SequoiaDB will also be shown in Notice later.

4) Query

foo=# select * from test;

5) Write Data

foo=# insert into test values('one',3);

6) Change Data

foo=# update test set id=9 where name='one';

7) Show all Tables

foo=# \d

8) View Description of the Table

foo=# \d test

9) Delete Table

foo=# drop foreign table test;

10) Exit PostgreSQL Shell

foo=# \q

Things to Pay Attention When Using

1) Mapping of Data Type

SequoiaDB PostgreSQL
string text
int/double/long numeric
date date
timestamp timestamp
array TYPE[]
boolean boolean
null text

2) Notice

2.1) Case Sensitivity of letters

Collection Space, Collection and Field name in SequoiaDB are case sensitive.

2.1.1) Collection and Collection space

For example, in SequoiaDB, there is a collection space named TEST, and a collection named CHEN, the mapping type in PostgreSQL will be:

foo=# create foreign table sdb_upcase_cs_cl (name text) server sdb_server options ( collectionspace 'TEST', collection 'CHEN' ) ;

2.1.2) Field Name

For example, in SequoiaDB, there is a collection space named foo, and a collection named bar, and they contain data of:

{
 "_id": {
   "$oid":"53a2a0e100e75e2c53000006"
 },
 "NAME": "test"
 }

Mapping type in PostgreSQL will be :

foo=# create foreign table sdb_upcase_field ("NAME" text) server sdb_server options ( collectionspace 'foo', collection 'bar' ) ;

Execute the query command:

foo=# select * from sdb_upcase_field;

Result is:

NAME
------
 test
 (1 rows)

2.2) Mapping the Data Type in SequoiaDB

For example, in SequoiaDB, there is a collection space named foo, and a collection named bar, and they contain data of:

{
 "_id": {
    "$oid":"53a2de926b4715450a000001"
 },
 "name": [
   1,
   2,
   3
 ],
 "id": 123
 }

The mapping table in PostgreSQL will be:

foo=# create foreign table bartest (name numeric[], id numeric) server sdb_server options ( collectionspace 'foo', collection 'bar' ) ;

Execute the query command

foo=# select * from bartest;

Result is

 name  | id
---------+-----
{1,2,3} | 123

2.3) Connecting SequoiaDB's Coord Node error

When SequoiaDB's coord node that PostgreSQL connecting re-starts, and the error is 

ERROR: Unable to get collection "chen.test", rc = -15
HINT: Make sure the collectionspace and collection exist on the remote database

Solution:

Exit PostgreSQL shell

foo=# \q

Re-enter PostgreSQL shell

$> bin/psql -p 5432 foo

Edit the Configuration File of PostgreSQL

1) View the default configuration in pg_shell:

foo=#\set
AUTOCOMMIT = 'on'
PROMPT1 = '%/%R%# '
PROMPT2 = '%/%R%# '
PROMPT3 = '>> '
VERBOSITY = 'default'
VERSION = 'PostgreSQL 9.3.4 on x86_64-unknown-linux-gnu, compiled by gcc (SUSE Linux) 4.3.4 [gcc-4_3-branch revision 152973], 64-bit'
DBNAME = 'foo'
USER = 'sdbadmin'
PORT = '5432'
ENCODING = 'UTF8'

2) Change the number of record to get each time doing pg_shell:

foo=#\set FETCH_COUNT 100
Note:

Change to getting 100 records each time doing the pg_shell.

Edit the configuration file in pg_shell directly, and the configuration will only activate in current pg_shell. You need to do the configuration again if you re-start the pg_shell.

3) Change the configuration file, get the number of records getting each time.

$> ${PG_HOME}/bin/pg_config -sysconfdir

Result is:

/opt/sequoiadb/pgsql/etc
Note:

if the directory to show is not exist, you may create it manually.

$> mkdir -p /opt/sequoiadb/pgsql/etc

Write the parameters you want to modify into the configuration file:

$>echo "\\set FETCH_COUNT 100" >> /opt/sequoiadb/pgsql/etc/psqlrc

4) Change the logs' level of pg_shell:

$>sed -i 's/#client_min_messages = notice/client_min_messages = debug1/g' pg_data/postgresql.conf

5) Change the logs' level of pg engine:

$>sed -i 's/#log_min_messages = warning/log_min_messages = debug1/g' pg_data/postgresql.conf
