Deployment

Installing PostgreSQL

1) Using source code to compile PostgreSQL

Download link:http://www.postgresql.org/ftp/source/

Compile and install after de-compressing (Need root privileges)

$> tar -zxvf postgresql-9.3.4.tar.gz
$> cd postgresql-9.3.4/
$> ./configure && make && make install

2) Switch user:

$>su - sdbadmin

3) Copy PostgreSQL file

$>cp -rf /usr/local/pgsql ~/

4) Enter PG's directory

$>cd pgsql

5) Add PostgreSQL's lib library in environment variables

$>export LD_LIBRARY_PATH=$(pwd)/lib:${LD_LIBRARY_PATH}
Note:

We suggest that users put PostgreSQL'slib to environment variables of sdbadmin, or you may need to add PostgreSQL'slib manually into LD_LIBRARY_PATH every time you login as sdbadmin

6) Create PostgreSQL's data directory

$>mkdir pg_data

7) Initializing the data directory(Can only be operate one time!)

$>bin/initdb -D pg_data/

Install SequoiaDB-PostgreSQL plug-in

1) Create PostgreSQL's lib directory

Get PostgreSQL'slibdir path

$> PGLIBDIR=$(bin/pg_config --libdir)

If the libdir directory is still not exist, you need to create the directory manually.

$> mkdir -p ${PGLIBDIR}

2) Create PostgreSQL's extension directory

Get PostgreSQL'ssharedir

$> PGSHAREDIR=$(bin/pg_config --sharedir)

Create extension directory in the sharedir directory

$> mkdir -p ${PGSHAREDIR}/extension

3) Copy PostgreSQL's extension file in SequoiaDB's installing package

After SequoiaDB installed, copy sdb_fdw.so from directory postgresql to PostgreSQL'slib directory. SequoiaDB's default installing path is /opt/sequoiadb

$> cp -f /opt/sequoiadb/postgresql/sdb_fdw.so ${PGLIBDIR}

4) Copy scripts of sdb_fdw.control and sdb_fdw-1.0.sql to directory extension, these two scripts need to be edited manually.

$> cp -f /opt/sequoiadb/postgresql/sdb_fdw.control ${PGSHAREDIR}/extension/ ;
$> cp -f /opt/sequoiadb/postgresql/sdb_fdw--1.0.sql ${PGSHAREDIR}/extension/ ;

sdb_fdw.control's script:

# sdb_fdw extension
comment = 'foreign data wrapper for SequoiaDB access'
default_version = '1.0'
module_pathname = '$libdir/sdb_fdw'
relocatable = true

sdb_fdw--1.0.sql's script:

/* contrib/sdb_fdw/sdb_fdw--1.0.sql */
-- complain if script is sourced in psql, rather than via CREATE EXTENSION
\echo Use "CREATE EXTENSION sdb_fdw" to load this file. \quit
CREATE FUNCTION sdb_fdw_handler()
RETURNS fdw_handler
AS 'MODULE_PATHNAME'
LANGUAGE C STRICT;
CREATE FUNCTION sdb_fdw_validator(text[], oid)
RETURNS void
AS 'MODULE_PATHNAME'
LANGUAGE C STRICT;
CREATE FOREIGN DATA WRAPPER sdb_fdw
HANDLER sdb_fdw_handler
VALIDATOR sdb_fdw_validator;

Deploy PostgreSQL

1) Check whether the port is occupied

PostgreSQL's default start port is '5432', check if the port is available (we suggest inspecting in root mode, and do the rest operations in sdbadmin still)

$>netstat -nap | grep 5432

If port 5432 is occupied, or user want to change the port, please execute:

$> sed -i "s/#port = 5432/port = 11780/g" pg_data/postgresql.conf

2) Start the PostgreSQL service process(must use sdbadmin to execute)

$> bin/postgres -D pg_data/ >> logfile 2>&1 &

3) Check if PostgreSQL start successfully

$> netstat -nap | grep 5432

Result is:

tcp   0   0 127.0.0.1:5432     0.0.0.0:*         LISTEN     20502/postgres
unix  2   [ ACC ]   STREAM    LISTENING   40776754 20502/postgres     /tmp/.s.PGSQL.5432

4) Create database of PostgreSQL

$> bin/createdb -p 5432 foo

Enter PostgreSQL's shell environment

$> bin/psql -p 5432 foo
