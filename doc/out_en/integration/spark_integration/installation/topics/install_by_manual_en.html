Install as standalone package

Spark Installation

Installing Apache Spark without SequoiaDB Administration Console is also straightforward.

First of all, you need to get the latest copy of the Spark framework. To do so, you can get the latest release here.

Taking example of Spark 1.3 with Hadoop 2.4, the download package will be called as "spark-1.3.tgz".

The package can be extracted by the following command:

$ tar –zxvf spark-1.3.tgz –directory /opt/

Once the package is extracted, you can cd to the directory and run a simple test using spark-shell by:

$ cd /opt/spark-1.3
$ bin/spark-shell 2>/dev/null

In order to build a high available cluster, there are at least 3 physical hosts required in the cluster. In this example we use name "server1", "server2" and "server3".

Download and extraction need to be performed on each of the box in the cluster.

Configure Spark cluster in Standalone mode

Once Spark binary image is installed, say /opt/spark-1.3, several simple steps need to be performed in order to build a high available cluster.

1) Download and configure Apache ZooKeeper

Apache ZooKeeper is a centralized service for maintaining configuration information, naming, providing distributed synchronization, and providing group services. All of these kinds of services are used in some form or another by distributed applications.

The download page for ZooKeeper can be found here

You need to select at least 3 nodes to make a high-available ZooKeeper cluster.

Once a package is downloaded on each box, it can be extracted by the following command:

$ tar –zxvf zookeeper-3.4.6.tar.gz –directory /opt/

2) Modify ZooKeeper configuration

$ cd /opt/zookeeper-3.4.6/conf
$ cp zoo_sample.cfg zoo.cfg
$ vi zoo.cfg

Add the following 3 lines and replace hostname to your real host names:

server.1=server1:2888:3888
server.2=server2:2888:3888
server.3=server3:2888:3888

And make sure change "dataDir" to somewhere other than /tmp directory, for example "/data/zookeeper".

Next, create myid file on each server. In this example, you can run the following commands against each server:

Server1: echo "1" > /data/zookeeper/myid
Server2: echo "2" > /data/zookeeper/myid
Server3: echo "3" > /data/zookeeper/myid

Note that the id number must be corresponding to the server.x configuration in zoo.cfg.

3) Start ZooKeeper service

The following command can be run on each host with ZooKeeper service configured:

$ bin/zkServer.sh start

You should start ZK service on all 3 nodes and you can also use the following command to check the status of ZooKeeper service:

$ bin/zkServer.sh status

One of the nodes should show "Mode: leader", and rest of the nodes should show "Mode: follower".

4) Configure Apache Spark in Standalone mode

Apache Spark can run in different modes. Standalone mode is recommended when working with SequoiaDB cluster.

Since local data access is always preferable than remote data lookup, it is recommended to install Apache Spark on all nodes in SequoiaDB database cluster.

Apache Spark standalone deployment looks like following:

There could be more than one Master nodes created in Spark cluster. Only one of them is in "Primary" node and rest of them are in "Standby" mode. ZooKeeper is used to keep track of each Master node to make sure there's always one Primary node at any given time.

It's recommended to run one Worker node on each of the box in SequoiaDB cluster.

Apache Spark configuration file is located at conf/spark-env.sh by default. Assume SequoiaDB is installed at /opt/sequoiadb, you can make a copy from spark-env.sh.template and make the following changes:

SPARK_DAEMON_JAVA_OPTS="-Dspark.deploy.recoveryMode=ZOOKEEPER -Dspark.deploy.zookeeper.url=server1:2181,server2:2181,server3:2181 -Dspark.deploy.zookeeper.dir=/data/zookeeper"
SPARK_WORKER_DIR=/data/spark_works
SPARK_LOCAL_DIRS=/data/spark_data1,/data/spark_data2
SPARK_CLASSPATH="/opt/sequoiadb/driver/java/sequoiadb-driver-1.12.jar:/opt/sequoiadb/driver/spark/spark-sequoiadb_2.10-1.12.0.jar"

The following configuration parameters are optional; it should be set based on hardware and workload. SequoiaDB automatically tunes those parameters if it’s installed by SequoiaDB Administration Center.

SPARK_WORKER_MEMORY

5) Start Spark master nodes

If you plan to host Master nodes on serverA and serverB, the following command can be used on those hosts:

$ cd /opt/spark-1.3
$ sbin/start-master.sh

6) Start Spark worker nodes

Since it’s recommended to run Worker node on all hosts in SequoiaDB cluster, you can run the following command in each box to start Worker node:

$ cd /opt/spark-1.3
$ nohup bin/spark-class org.apache.spark.deploy.worker.Worker spark://serverA:7077,serverB:7077>logs/worker.out &
