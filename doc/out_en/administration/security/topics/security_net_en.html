Network Security

SequoiaDB supports SSL by setting. SequoiaDB client and SequoiaDB instance can use SSL to encrypt the connection directly.

SequoiaDB Security Support

If you want to use SSL encrypted connection, you need to update to SequoiaDB 1.12 or later versions.

And this feature is only available for enterprises users, not included in community version.

Client Support

All official supported client will support SSL, including C, C++, Java, Python, C#, PHP, REST API and SDB Shell.

Configure SequoiaDB to Use SSL

When installing and deploying, activate SequoiaDB SSL supporting by modify parameter:

--usessl, Default value is false, activate SSL with value of true. Then it will allow client connecting with SSL, and also accept non-SSL connections at the same time.

sequoiadb --usessl=true

Also, it can use configuration file to configure, see in database configuration.

SequoiaDB will automatically create certification under SSL mode.

Client Using SSL

Client must incorporate with SequoiaDB under SSL mode to use SSL. All official supported client will support SSL.

  • C

    C driver port uses sdbSecureConnect() and sdbSecureConnect1() to create SSL connection, operation is like sdbConnect() and sdbConenct1()

    See in C Driver。

  • C++

    In C++ driver class, sdb constructor has parameter useSSL, when it equals true, SSL is activated.

    See in C++ C++ Driver。

  • Java

    In Java driver class, com.sequoiadb.net.ConfigOptionshas interface setUseSSL(Boolean useSSL). When it is true, it use SSL connection.

    See in Java Driver。

  • Python

    In Python driver client constructor has a optional parameter ssl, when set as true, SSL will be activated.

    See in Python Driver。

  • C#

    In C# driver class, SequoiaDB.ConfigOptions has property of UseSSL, set as true to use SSL connection.

    See in C# Driver。

  • PHP

    In PHP driver, there is a class named SecureSdb which is sub-class of SequoiaDB, object of SecureSdb uses SSL to connect.

    See in PHP Driver。

  • REST API

    REST API supports https.

    See in REST API。

  • Sdb shell

    Sdb shell has a shared class SecureSdb, and it is sub-class of Sdb。Object of SecureSdb uses SSL to connect.

    See in SequoiaDB shell。

SequoiaDB Tools Support

Sdbexprt,sdbimprt, sdblobtool, sdbtop supports SSL connection, see in DataBase Tool。
