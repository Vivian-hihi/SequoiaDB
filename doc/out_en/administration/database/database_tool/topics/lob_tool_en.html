Large Object(LOB) Tool — sdblobtool

Sdblobtool is a tool for managing LOBs.

Functions List

Function Description
Export Export LOB of the collectioninto local file
Import Import LOB in local file into collection
Move Move a LOB from one collection to another

Options

Export options

Name Description Default Value Required
hostname Coord address localhost No
svcname Coord service name 11810 No
usrname user name - No
passwd password - No
operation operation's type - yes(export)
file local file's path - yes
collection full name of the collection to export - yes
ssl use SSL connect false No

Import options

Name Description Default Value Required
hostname Coord address localhost No
svcname Coord service name 11810 No
usrname user name - No
passwd password - No
operation operation's type - yes(export)
file local file's path - yes
collection full name of the collection to import - yes
ignorefe if this LOB has already existed, ignore this error and start import the next record. false No
ssl use SSL connect false No
Note:
  • When you need to activate ignorefe, you only need to add –ignorefe, not need to give a value. The rest are the same.
  • Local file must be created by other exporting.(can't manually create).

Migration options:

Name Description Default Value Required
hostname Coord address localhost No
svcname Coord service name 11810 No
usrname user name - No
passwd password - No
operation operation's type - yes(migration)
file local file's path - yes
collection full name of the collection to import - yes
ignorefe if this LOB has already existed, ignore this error and start import the next record. false No
dsthost destination Coord address localhost No
dstservice destination service name 11810 No
dstusrname destination user name - No
dstpasswd destination password - No
dstcollection destination collection's full name - Yes
ssl use SSL connect false No

Log

When using sdblobtool, the log will be created under current running directory in the log file of "sdblobtool.log". When there is error, the running record can be view in the log.

Common Errors

Error Code(rc) Description Solutions
-5 local file already exists or there is LOBs of same OID in collection 1. If doing exporting, check whether the local file existing already.
2. If dong importing or moving operations, check if destination collection contains LOB with same OID.
-6 Illegal Parameter Check the parameters, see if there is any input error.
-10 System Error Check the error according to log created.
-15 Can not connect to the destination 1. Check the address parameters' correctness.
2. Check if the database start correctly.
3. If use hostname as a parameter, check whether the local hostname list is configured correctly.
4. Check if the firewall is on.
-23 The collection is not exist 1. Check whether parameters related to collection are correct.
2. Check whether the collection exist.

Example

Export LOB of collection foo.bar into local file mylob:
./sdblobtool --operation export --hostname 192.168.1.1 --svcname 11810 --collection foo.bar --file /opt/mylob
Import LOB in local file mylob to collection foo.bar, when occur the existing LOB, it will automatically skip.
./sdblobtool --operation import --hostname 192.168.1.1 --svcname 11810 --collection foo.bar --file /opt/mylob --ignorefe
Migrate the LOB in one collection to another:
./sdblobtool --operation migration --hostname 192.168.1.1 --svcname 11810 --collection foo.bar --dsthost 192.168.1.2 --dstservice 11810 --dstcollection foo.bar1
