数据备份与恢复 / 数据恢复

例程

  1. 停止被恢复的复制组 
    $ /opt/sequoiadb/bin/sdb 
>db=new Sdb('192.168.20.35',11810)
192.168.20.35:11810
>shard=db.getShard('foo')
foo
>shard.stop()
  2. 恢复配置文件 
    $ cp vmsvr1-rhel-x64.11820.sdb.conf /opt/sequoiadb/conf/local/11820/sdb.conf
  3. 恢复数据文件 
    $ tar -zxvf vmsvr1-rhel-x64.11820.2013_08_21.tar.gz
/tmpfs/data1/foo.dat
/tmpfs/data1/one.dat
/tmpfs/data1/SYSTEMP.dat
/tmpfs/data1/foo.idx
/tmpfs/data1/one. idx
/tmpfs/data1/SYSTEMP.idx
  4. 删除日志文件，日志文件存在于配置文件的logpath目录，默认情况下在 <数据库目录 >/replicalog中 
    $ rm -rf /tmpfs/data1/replicalog
  5. 启动复制组 
    $ /opt/sequoiadb/bin/sdb 
>db=new Sdb('192.168.20.35',11810)
192.168.20.35:11810
>shard=db.getShard('foo')
foo
>shard.start()
Note: 仅恢复复制组内某一个数据节点会造成数据之间的不一致。
