集群模式

数据节点与编目节点

集群模式下，用户可以选择停止所有数据复制组后，将每一个复制组中的原主节点的数据库目录与索引目录中的所有文件归档。每个复制组中的主节点可以在编目信息的SYSCAT.SYSNODES集合中找到，为PrimaryNode字段。

如果用户希望在不中断服务的情况下备份，可以选择停止每个复制组中的一个备节点，并将该节点中的数据库目录与索引目录中的所有文件归档。备份完成后重新启动该节点，该节点会自动与组内其它节点同步。

在指定配置文件的情况下，数据库目录与索引目录为该节点对应的配置文件中的dbpath与indexpath所指定的目录。默认情况下indexpath与dbpath相同。

协调节点

由于协调节点不包含任何用户数据，因此不需要进行数据备份。
