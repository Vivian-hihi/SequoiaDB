数据迁移 — 导入

sdbimprt

sdbimprt是一个SequoiaDB数据库导入工具，它可以实现从JSON格式或CSV格式的数据存储文件导入到SequoiaDB数据库。

JSON格式的记录必须符合Json的定义，以左右花括号作为记录的分界符，并且字符串类型的数据必须包含在两个双引号之间，转义字符定义为反斜杠。

CSV为Comma-Separated Value格式，为逗号分割数值。默认情况下每条记录以0x0D分割，字段之间以逗号分割。用户能够指定字段之间的分隔符与记录之间的分隔符。 分隔符定义（只支持ASCII字符）：

用途 默认 可配
字符串分隔符 " 是
字符分隔符 , 是
记录分割符 '\n'（0X0D） 是

选项

参数 描述
--help 返回基本帮助和用法文本 。
--hostname,-h 从指定主机名的SequoiaDB中导入数据。默认情况下sdbexprt尝试连接到本地主机。
--svcname,-s 指定的端口号。默认情况下sdbexprt尝试连接到端口号11810的主机。
--type 指定的导入数据格式。默认csv，数据格式可以是csv或是json。
--file 指定要导入的文件名。
--delchar,-a 指定字符分隔符。默认是 "， csv格式有效。
--delfield,-e 指定字段分隔符。默认是 ,，csv格式有效。
--delrecord,-r 指定记录分隔符。默认是 \n，csv格式有效。
--csname,-c 指定导入数据的集合空间名。
--clname,-l 指定导入数据的的集合名。
--fields 指定导入数据的的字段名。csv格式有效。
--headerline 指定导入数据首行是否作为字段名，默认false，csv格式有效。
--sparse 指定导入数据时，自动添加字段名，默认true，csv格式有效。
--extra 指定导入数据时，自动添加值，默认false，csv格式有效。
--linepriority 当前分隔符默认的优先级为：记录分隔符，字符分隔符，字段分隔符，默认值是true；如果设置为false，那么分隔符的优先级为：字符分隔符，记录分隔符，字段分隔符。
Note: Linepriority参数需要被特别关注，如果设置不当，可能会导入数据失败。
当记录中包含“记录分隔符”并且linepriority为true时，工具会优先按照“记录分隔符”解析，而导致导入失败。比如：如果记录为 {"name": "Mike\n"}，应当选择linepriority为false模式。

用法

在下面的例子，sdbimprt将数据导入到本地数据库端口11810中对应集合空间foo和集合bar，导入类型是csv，导入文件为test.csv

sdbimprt -h localhost -s 11810 --type csv --file test.csv -c foo -l bar

sdbload

sdbload是一个SequoiaDB数据库批量导入工具，它可以实现从JSON和CSV格式的数据存储文件快速导入到SequoiaDB数据库。

JSON格式的记录必须符合Json的定义，以左右花括号作为记录的分界符，并且字符串类型的数据必须包含在两个双引号之间，转义字符定义为反斜杠。

选项

参数 描述
--help 返回基本帮助和用法文本。
--hostname,-h 从指定主机名的 SequoiaDB 中导入数据。默认情况下 sdbload 尝试连接到本地主机。
--svcname,-s 指定的端口号。默认情况下sdbload尝试连接到端口号11810的主机。
--delchar,-a 指定字符分隔符。默认是 "，csv格式有效。
--delfield,-e 指定字段分隔符。默认是 ,，csv格式有效。
--delrecord,-r 指定记录分隔符。默认是 \n，csv格式有效。
--csname,-c 指定导入数据的集合空间名。
--clname,-l 指定导入数据的的集合名。
--insertnum,-n 指定每一批导入的记录数。默认是100。
--file 指定导入文件名。
--model 指定获取数据模式，默认 io ，模式可以是 io 或 hdfs。
--libpath 指定加载库的路径，hdfs模式下有效。
--sourcehost 指定连接数据源服务端的主机名，hdfs模式有效。
--sourceport 指定连接数据源服务端的端口，hdfs模式有效。
--sourceuser 指定连接数据源服务端的用户名，hdfs模式有效。
--type 指定导入数据的文件类型，默认csv，类型可以是json或csv。
--fields 指定导入数据的字段名，csv格式有效。
--headerline 指定导入数据首行是否作为字段名，默认false，csv格式有效。
--sparse 指定导入数据时，自动添加字段名，默认true，csv格式有效。
--extra 指定导入数据时，自动添加值，默认false，csv格式有效。
--linepriority 当前分隔符默认的优先级为：记录分隔符>字符分隔符>字段分隔符，默认值是true；如果设置为false，那么分隔符的优先级为：字符分隔符>记录分隔符>字段分隔符。

用法

在下面的例子，sdbload将数据导入到端口11810中对应集合空间foo和集合bar，导入文件为test.json。 

sdbload –h localhost -s 11810 --file test.json -c foo -l bar –type json
