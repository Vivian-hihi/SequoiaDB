##NAME##

enableReadonly - enable readonly mode

##SYNOPSIS##

**SdbDC.enableReadonly()**

##CATEGORY##

SdbDC

##DESCRIPTION##

This function is used to enable readonly mode for the cluster. After enabling, the cluster will only allow read operations and reject all write operations.

##PARAMETERS##

None

##RETURN VALUE##

When the function executes successfully, there is no return value.

When the function fails, an exception will be thrown and an error message will be printed.

##ERRORS##

The common exceptions of `enableReadonly()` function are as follows:

| Error Code | Error Type | Description | Solution |
| ---------- | ---------- | ----------- | -------- |
| -6 | SDB_INVALIDARG | Parameter type error | Check whether the parameter type is correct |

When the exception happens, use [getLastErrMsg()][getLastErrMsg] to get the error message or use [getLastError()][getLastError] to get the [error code][error_code]. For more details, refer to [Troubleshooting][faq].

##VERSION##

v3.0 and above

##EXAMPLES##

**Example 1:** Enable readonly mode for cluster

```lang-javascript
> var dc = db.getDC()
> dc.enableReadonly()
```
