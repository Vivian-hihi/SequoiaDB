##NAME##

createAutoIncrement - Create one or more auto increment fields in a specified collection.

##SYNOPSIS##

**db.collectionspace.collection.createAutoIncrement(<option|options>)**

##CATEGORY##

Collection

##DESCRIPTION##
Create one or more auto increment fields in a specified collection.

##PARAMETERS##

* `option|options` ( *Json Object | Array of Json Objects*, *Required* )

   Auto increment field parameters.

1. `Field` ( *String*, *Required* )

   Auto increment field name. It cannot contain invisible characters and start with "$" or blank character.

2. `Increment` ( *Int32*, *Optional* )

   Each increase in the interval. The default value is 1. It can be positive or negative integers, but cannot be 0.

3. `StartValue` ( *Int64*, *Optional* )

   The starting value of the auto increment field. The default value is 1.

4. `MinValue` ( *Int64*, *Optional* )

   The minimum value of the auto increment field. The default value is 1.

5. `MaxValue` ( *Int64*, *Optional* )

   The maximum value of the auto increment field. The default value is 2^63-1.

6. `CacheSize` ( *Int32*, *Optional* )

   The number of sequence values that the catalog node caches each time. The default value is 1000. It must be greater than zero.

7. `AcquireSize` ( *Int32*, *Optional* )

   The number of sequence values that the coordinate node acquires each time. The default value is 1000. It must be less than or equal to CacheSize.

8. `Cycled` ( *Bool*, *Optional* )

   Whether loops are allowed when sequence values reach a maximum or minimum value. Its default value is false.

9. `Generated` ( *String*, *Optional* )

   Auto increment field generation method. The defalut value is "default". Can be the follow values:

   * "always": Auto increment fields are always generated by the server, ignoring client's settings.
   * "default": Allow client's settings.
   * "strict": Allow the client to set and adding type detection. Error reporting when the type is not numeric.

**Note:**

1. Auto increment field can be a nested field.
2. When Increment is a positive integer, StartValue defaults to 1, MinValue defaults to 1, and MaxValue defaults to 2^63-1. When Increment is a negative integer, StartValue defaults to -1, MinValue defaults to -2^63, and MaxValue defaults to -1.
3. StartValue must be in the [MinValue, MaxValue] interval.
4. Sequence values are strictly incremented when set to 1 by AcquireSize. CacheSize and AcquireSize are performance-related parameters that are recommended for careful modification.
5. When a main-collection is used, only the auto increment field on main-collection takes effect, and the auto increment field on sub-collection is invalid.
6. If the client sets the value of the auto increment field, or modifies the property of the auto increment field during use, the field values may not be unique. If you want to ensure that the modified value is unique, it is recommended to use a unique index.
7. Individual nodes do not support auto increment fields.

##RETURN VALUE##

On success, return void.

On error, exception will be thrown and ouput the error message.

##EXAMPLES##

1. Create an auto increment field whose vlaue is always generated by the server, ignoring the client's settings.

   ```lang-javascript
   > db.foo.bar.createAutoIncrement( { Field: "studentID", Generated: "always" } )
	```

2. Create two auto increment fields.

   ```lang-javascript
   > db.foo.bar.createAutoIncrement( [ { Field: "comID" }, { Field: "innerID" } ] )
	```
